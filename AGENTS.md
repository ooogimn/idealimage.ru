# AGENTS RULES (UNIVERSAL TEMPLATE)

## 1) Разрешение на Shell
- Агенту разрешено выполнять терминальные команды в этой среде.
- До явного разрешения пользователя (`делай`, `запускай`, `разрешаю`) команды не выполнять.
- После разрешения выполнять команды самостоятельно до закрытия задачи.

## 2) Безопасность
- Не выполнять разрушительные команды без отдельного подтверждения (`reset --hard`, удаление данных, force push).
- Перед запуском долгих процессов проверять, не запущены ли они уже.
- При ошибке окружения сначала диагностировать (WSL/Docker/Python), затем продолжать.

## 3) Работа с проектом
- Предпочитать правки на локальной машине; серверные действия делать только при явном запросе.
- После изменений проверять результат (миграции, запуск приложения, ключевые smoke-checks).
- Если найдена блокирующая проблема, кратко сообщить причину и предложить следующий точный шаг.

## 4) Формат взаимодействия
- Кратко сообщать: что делается сейчас, что сделано, что осталось.
- Не спрашивать лишнего, если данных достаточно для выполнения задачи.
